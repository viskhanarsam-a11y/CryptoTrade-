# CryptoTrade Spec Guardian

Этот скилл нужен, чтобы Codex применял главный файл правил проекта и находил нужные спеки CryptoTrade.

## Главный файл правил проекта

`Agent.md` — главный файл правил проекта для этого скилла. Перед любой работой по CryptoTrade обязательно прочитай `.codex/skills/crypto-trade-spec-guardian/Agent.md` и следуй ему.

`SKILL.md` нужен, чтобы Codex понял, когда включать этот скилл и какие файлы читать. `Agent.md` определяет главные команды проекта.

Если `Agent.md` и другой файл скилла расходятся, следуй `Agent.md` и сообщи о конфликте.

## Основной порядок работы

1. Найди корень проекта, где есть папка `Spec/`.
2. Прочитай `.codex/skills/crypto-trade-spec-guardian/Agent.md`, потому что это главный файл правил проекта.
3. По команде пользователя сам определи область задачи и какие спеки могут быть нужны.
4. Не читай все спеки подряд. Читай только те файлы, которые относятся к задаче.
5. Используй `Spec/1 Functional.md` как главный рабочий ориентир проекта.
6. Читай `Spec/Global.spec.md` только когда нужно проверить смысл продукта, аудиторию или границы проекта.
7. Используй `references/spec-map.md`, названия файлов и поиск по `Spec/`, чтобы найти релевантные `feature-specs`, `technical-specs` и `user-stories`.
8. Если задача выходит за рамки спек, требование отсутствует или спеки конфликтуют, остановись и сообщи пользователю.
9. Реализуй, проверь или напиши спеку строго по выбранным требованиям.
10. В финальном ответе используй формат из `Agent.md`.

## Файлы скилла

- `Agent.md` — главные команды проекта.
- `references/spec-map.md` — карта, какие спеки читать под задачу.
- `agents/openai.yaml` — служебное описание скилла для интерфейса.
