# Feature spec: Регистрация и вход

## 1. Назначение

Регистрация и вход нужны, чтобы пользователь мог сохранять свои данные и настройки.

Эта feature опирается на `Spec/1 Functional.md`, раздел "Регистрация и вход".

## 2. Что пользователь может делать

Пользователь должен иметь возможность:
- создать аккаунт;
- войти в аккаунт;
- выйти из аккаунта.

## 3. Что нужно сохранять после входа

Регистрация нужна, чтобы в будущем сохранять:
- прогресс мини-обучения;
- избранные монеты;
- пользовательские настройки;
- настройки спотового графика;
- выбранный язык.

## 4. Основной сценарий регистрации

Пользователь:
- открывает регистрацию;
- вводит данные;
- создает аккаунт;
- видит, что он вошел или может войти.

## 5. Основной сценарий входа

Пользователь:
- открывает вход;
- вводит данные;
- входит в аккаунт;
- видит состояние авторизации.

## 6. Правила

Пользователь должен понимать, вошел он в аккаунт или нет.

После выхода сайт должен показывать состояние гостя.

Ошибки регистрации и входа должны быть понятными.

## 7. Что проверить

Нужно проверить:
- пользователь может открыть регистрацию;
- пользователь может создать аккаунт;
- пользователь может открыть вход;
- пользователь может войти;
- пользователь может выйти;
- ошибки показываются понятным текстом;
- после входа доступны данные, которые должны сохраняться в будущем.
