# Technical spec: Frontend, навигация и состояния

## 1. Назначение

Эта спека описывает маршруты, состояние интерфейса и общие frontend-правила.

## 2. Маршруты

Минимальные маршруты:
- `/` - главная страница;
- `/learn` - мини-обучение;
- `/learn/:lessonId` - урок;
- `/top-coins` - топ монеты;
- `/coins/:coinId` - страница монеты;
- `/check` - проверка монет;
- `/spot` - спотовый график;
- `/login` - вход;
- `/register` - регистрация.

## 3. Глобальные состояния

Нужно поддерживать:
- выбранный язык;
- состояние авторизации;
- mobile navigation state;
- loading/error для глобальных данных;
- выбранную тему оформления, если она будет добавлена.

## 4. Feature-состояния

Для каждой feature должны быть состояния:
- initial;
- loading;
- success;
- empty;
- error.

UI не должен зависеть от одного счастливого сценария.

## 5. Ошибки

Ошибки должны быть:
- понятными;
- короткими;
- одинаковыми по стилю;
- без технических stack trace для пользователя.

Для разработчика техническая ошибка должна логироваться отдельно.

## 6. Mobile

На mobile нужно проверить:
- отсутствие горизонтального скролла;
- читаемость таблиц;
- доступность основных кнопок;
- поведение меню;
- полноэкранный график;
- формы входа и регистрации.

## 7. Что надо определить глубже

Нужно уточнить:
- точный роутинг;
- структуру layout;
- какие состояния управляются глобально;
- какую state management библиотеку использовать;
- дизайн error/loading компонентов.
