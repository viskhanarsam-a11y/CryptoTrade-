# Technical spec: Авторизация и хранение данных

## 1. Назначение

Эта спека описывает регистрацию, вход, гостевой режим и сохранение пользовательских данных.

## 2. Типы пользователей

В системе есть два состояния:
- гость;
- зарегистрированный пользователь.

Гость может пользоваться основными разделами сайта, но его данные хранятся локально.

Зарегистрированный пользователь может сохранять данные в аккаунте.

## 3. Что хранить локально

Для гостя можно хранить:
- выбранный язык;
- последний результат проверки монеты;
- настройки графика;
- выбранную последнюю монету;
- согласие с дисклеймером, если оно будет добавлено.

Локальные данные не должны считаться надежным источником истины.

## 4. Что хранить в аккаунте

Для зарегистрированного пользователя нужно хранить:
- email или внешний auth id;
- выбранный язык;
- избранные монеты;
- прогресс уроков;
- настройки графика;
- дату создания аккаунта;
- дату последнего входа.

## 5. Перенос данных после входа

Если пользователь был гостем и затем вошел:
- локальный язык можно применить к аккаунту;
- локальные настройки графика можно предложить перенести;
- избранные монеты нужно объединять без дублей;
- прогресс уроков не должен затираться меньшим прогрессом;
- серверные данные не должны неожиданно перезаписываться локальными.

## 6. Минимальная модель данных

`User`
- `id`;
- `email`;
- `createdAt`;
- `lastLoginAt`.

`UserSettings`
- `userId`;
- `language`;
- `chartSettings`.

`WatchlistItem`
- `userId`;
- `coinId`;
- `createdAt`.

`LessonProgress`
- `userId`;
- `lessonId`;
- `status`;
- `completedAt`.

## 7. Что надо определить глубже

Нужно выбрать:
- способ авторизации;
- требования к паролю;
- нужна ли email-верификация;
- нужна ли авторизация через Google;
- где хранить сессии;
- точные правила merge локальных и серверных данных.
