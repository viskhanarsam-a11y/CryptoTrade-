# Technical spec: Источники данных и API

## 1. Назначение

Эта спека описывает, откуда CryptoTrade берет данные о монетах, ценах, графиках и халяль-проверке.

## 2. Типы данных

Проекту нужны данные:
- список топ-20 монет;
- текущая цена;
- изменение цены;
- свечные данные для графика;
- базовая информация о монете;
- халяль-статус;
- объяснение статуса;
- источники и риски.

## 3. Рыночные данные

Для MVP нужно выбрать один основной источник рыночных данных.

Возможные варианты:
- CoinGecko API;
- CoinMarketCap API;
- Binance API для торговых пар и свечей;
- TradingView widget для графиков без собственного хранения свечей.

Для каждой монеты нужно хранить внутренний `coinId`, `symbol`, `name` и внешний идентификатор источника данных.

## 4. API-контракты

Минимальные endpoints или client methods:

`getTopCoins()`
- возвращает топ-20 монет;
- включает название, тикер, цену, изменение цены и внешний id.

`searchCoins(query)`
- ищет по названию и тикеру;
- возвращает список совпадений.

`getCoinDetails(coinId)`
- возвращает подробную информацию о монете.

`getHalalCheck(coinId)`
- возвращает статус, объяснение, риски и источники.

`getChartData(coinId, timeframe)`
- возвращает свечи, если проект использует собственную библиотеку графиков.

## 5. Кэширование

Текущие цены можно кэшировать на короткое время.

Рекомендуемые правила:
- цены: 30-120 секунд;
- топ-20: 5-15 минут;
- данные монеты: 1-24 часа;
- халяль-статус: до следующего ручного или проверенного обновления;
- обучающий контент: до новой версии приложения.

## 6. Поведение при ошибках

Если внешний API недоступен:
- интерфейс не должен ломаться;
- пользователь видит понятное сообщение;
- старые кэшированные данные можно показывать с пометкой, что они могут быть неактуальны;
- халяль-статус нельзя менять на `Unknown` только из-за временной ошибки API.

## 7. Что надо определить глубже

Нужно выбрать:
- основной источник цен;
- источник свечей;
- список топ-20 монет;
- лимиты API;
- стратегию кэширования;
- fallback, если API недоступен;
- формат ошибок API.
