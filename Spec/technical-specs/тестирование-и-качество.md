# Technical spec: Тестирование и качество

## 1. Назначение

Эта спека описывает, что нужно проверять технически, чтобы CryptoTrade не ломался при изменениях.

## 2. Unit tests

Unit-тестами нужно покрыть:
- маппинг халяль-статусов в цвета;
- форматирование цен;
- форматирование процентов;
- merge локальных и серверных настроек;
- выбор языка;
- нормализацию данных монеты из внешнего API.

## 3. Integration tests

Integration-тестами нужно покрыть:
- поиск монет;
- получение топ-20;
- получение халяль-статуса;
- авторизацию;
- сохранение настроек;
- загрузку уроков.

## 4. E2E tests

E2E-тестами нужно покрыть сценарии:
- пользователь открывает главную и переходит в основные разделы;
- пользователь смотрит топ монеты;
- пользователь открывает график;
- пользователь включает и закрывает fullscreen-график;
- пользователь проверяет монету;
- пользователь меняет язык;
- пользователь регистрируется, входит и выходит.

## 5. Mobile checks

На mobile нужно проверить:
- нет горизонтального скролла;
- кнопки доступны для нажатия;
- график помещается;
- fullscreen закрывается;
- тексты не вылезают из блоков;
- формы не ломаются.

## 6. Критичные проверки

Критично проверить:
- `Halal-friendly` не отображается красным;
- `Not halal` не отображается зеленым;
- `Unknown` не выглядит как положительный статус;
- временная ошибка API не меняет халяль-статус монеты;
- fullscreen не включается автоматически после входа.

## 7. Что надо определить глубже

Нужно выбрать:
- test runner;
- e2e-инструмент;
- минимальный набор smoke-тестов;
- CI-процесс;
- правила перед релизом.
