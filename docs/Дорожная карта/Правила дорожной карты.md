# Правила дорожной карты

`docs/Дорожная карта` — это история развития проекта CryptoTrade по областям.

Она не заменяет спеки и не заменяет активные планы разработки.

## Когда обновлять

Дорожную карту нужно обновлять после того, как конкретный план разработки полностью завершен, проверен и перенесен из:

`docs/Планы разработки/Активные планы`

в:

`docs/Планы разработки/Завершенные планы`

## Как обновлять

1. Определи, какие области проекта затронул завершенный план.
2. Обнови только нужные файлы в `docs/Дорожная карта`.
3. Добавь запись в хронологическом порядке.
4. Не удаляй старые записи.
5. В записи укажи дату, название завершенного плана и коротко опиши, что было сделано.

## Формат записи

```md
## YYYY-MM-DD — Название завершенного плана

- Завершенный план: `docs/Планы разработки/Завершенные планы/имя-плана.md`
- Что изменилось:
  - ...
- Проверено:
  - ...
```

## Области

- `Общая хронология.md` — краткая общая история завершенных планов.
- `Данные.md` — источники данных, API, модели, хранение.
- `Фронтенд.md` — компоненты, страницы, навигация, состояния интерфейса.
- `Дизайн.md` — внешний вид, адаптивность, визуальные решения.
- `Качество.md` — тесты, проверки, баги, стабильность.
- `Безопасность и ограничения.md` — безопасность, правовые ограничения, границы продукта.
- `Деплой.md` — окружения, сборка, публикация.
