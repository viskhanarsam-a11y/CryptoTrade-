# CryptoTrade Functional Specification

## 1. Назначение функциональной спеки

Эта спека описывает, что пользователь может делать на сайте CryptoTrade.

Сайт подходит для пользователей, которые хотят:
- научиться трейдингу;
- искать халяльные монеты;
- смотреть графики монет;
- проходить раздел "Основы Трейдинга";
- проверять криптоактивы перед самостоятельным решением.

## 2. Основные разделы сайта

На сайте должны быть основные разделы:
- главная страница;
- Основы Трейдинга;
- топ монеты;
- проверка монет;
- спотовый график;
- регистрация и вход;
- переключение языка.

Навигация между разделами должна быть понятной на desktop и mobile.

## 3. Основы Трейдинга

Пользователь может открыть раздел "Основы Трейдинга" и увидеть список обучающих тем.

Темы раздела "Основы Трейдинга":
- ордер-блоки;
- синхронизация таймфреймов;
- слом структуры BOS;
- имбаланс;
- ликвидность.

Для каждой темы нужно показывать:
- название темы;
- короткое описание;
- обучающий материал;
- видео, если оно добавлено в проект;
- статус прохождения, если пользователь зарегистрирован.

Когда пользователь нажимает на тему, должна открываться страница или экран урока.

Урок может содержать:
- текстовое объяснение;
- видео;
- примеры на графике;
- короткий итог по теме.

## 4. Регистрация и вход

Пользователь может зарегистрироваться на сайте.

Регистрация нужна, чтобы в будущем сохранять:
- прогресс в разделе "Основы Трейдинга";
- избранные монеты;
- пользовательские настройки;
- настройки спотового графика;
- выбранный язык.

Пользователь должен иметь возможность:
- создать аккаунт;
- войти в аккаунт;
- выйти из аккаунта.

## 5. Переключение языка

На сайте должна быть возможность переключить язык:
- русский;
- английский.

Переключение языка должно менять тексты интерфейса, названия разделов, кнопки и основные сообщения сайта.

Выбранный язык должен сохраняться для пользователя. Если пользователь не зарегистрирован, язык можно хранить локально в браузере.

## 6. Топ монеты

В разделе "Топ монеты" пользователь видит список из 20 монет.

Список топ-20 монет задан в `docs/Spec/feature-specs/топ-монеты.md`.

Для каждой монеты в списке нужно показывать:
- название монеты;
- тикер;
- логотип монеты, если он доступен;
- текущую цену, если данные доступны;
- краткое изменение цены;
- кнопку или строку для открытия монеты.

Когда пользователь нажимает на монету, открывается экран с графиком этой монеты.

График должен быть похож на TradingView:
- большой рабочий график;
- свечи или линия цены;
- удобный масштаб;
- нормальная читаемость на desktop и mobile;
- возможность открыть график на весь экран.

Для MVP график монеты должен быть реализован через `TradingView widget`.

## 7. Проверка монет

В разделе "Проверка монет" пользователь может найти монету и проверить ее на халяльность.

Пользовательский сценарий:
- пользователь открывает раздел "Проверка монет";
- вводит название или тикер монеты;
- выбирает найденную монету;
- запускает проверку;
- видит результат проверки.

Результат проверки должен показывать:
- название монеты;
- статус халяльности;
- короткое объяснение статуса;
- возможные риски;
- источники данных, если они доступны.

Возможные статусы:
- `Halal-friendly`;
- `Not halal`;
- `Unknown`.

Цвета статусов:
- `Halal-friendly` показывается зеленым цветом;
- `Not halal` показывается красным цветом;
- `Unknown` показывается нейтральным серым цветом.

Если проверка занимает время, сайт должен показывать индикатор загрузки, чтобы пользователь понимал, что проверка выполняется.

Для статуса `Unknown` сайт должен показывать сообщение:
`Статус этой монеты пока неизвестен. Мы сообщим больше, когда появится проверенная информация.`


Сайт не должен путать цвет и статус:
- `Halal-friendly` всегда зеленый;
- `Not halal` всегда красный;
- `Unknown` всегда нейтральный.

Последний результат проверки можно временно сохранять, чтобы пользователь не потерял его при обновлении страницы.

Сайт не должен выдавать религиозное заключение. Он показывает предварительную оценку и объясняет, почему монета может попадать в конкретный статус.

## 8. Спотовый график

В разделе "Спотовый график" пользователь видит список монет, доступных для просмотра.
Пользовательский сценарий:
- пользователь открывает раздел "Спотовый график";
- нажимает на любую монету;
- открывается график выбранной монеты;
- пользователь смотрит график в обычном размере;
- пользователь может нажать кнопку полноэкранного графика;
- график открывается на весь экран.

График в этом разделе должен выглядеть как график на TradingView:
- крупная область графика;
- свечной вид, если он доступен;
- панель управления таймфреймом, если она будет реализована;
- удобное масштабирование;
- кнопка полноэкранного режима.

Для MVP спотовый график должен быть реализован через `TradingView widget`.

В обычном режиме график должен занимать нормальный размер экрана и не быть слишком маленьким.

Если пользователь вошел в аккаунт, сайт должен сохранять его настройки спотового графика и восстанавливать их при следующем входе.

К настройкам графика относятся:
- выбранная монета или торговая пара;
- таймфрейм;
- тип отображения графика, например свечи или линия;
- масштаб и положение графика, если это поддерживается;
- включенные индикаторы и инструменты, если они будут реализованы;
- полноэкранный режим не должен включаться автоматически после входа, но последняя рабочая конфигурация графика должна сохраняться.

Для выбранной монеты нужно показывать ее логотип, если он доступен. Если логотип недоступен или не загрузился, сайт показывает нейтральную заглушку с тикером монеты.

Если пользователь не вошел в аккаунт, настройки графика можно временно сохранять локально в браузере. После входа локальные настройки можно предложить применить к аккаунту или автоматически перенести, если это не перезаписывает уже сохраненные настройки пользователя.

## 9. Мобильная версия

Сайт обязательно должен хорошо выглядеть на мобильных устройствах.

Требования для mobile:
- все кнопки помещаются в экран;
- кнопки меньше среднего размера, но остаются удобными для нажатия;
- график помещается в экран и не ломает страницу;
- пользователь может открыть график на весь экран;
- меню и разделы удобно переключаются;
- текст не вылезает за пределы блоков;
- важные действия доступны без горизонтального скролла.

Особенно важно проверить на мобильном:
- раздел "Основы Трейдинга";
- раздел "Топ монеты";
- раздел "Проверка монет";
- раздел "Спотовый график";
- обычный и полноэкранный режим графика.

## 10. Главные пользовательские сценарии

### 10.1 Пользователь изучает трейдинг

Пользователь открывает раздел "Основы Трейдинга", выбирает тему, смотрит материал или видео и возвращается к списку уроков.

### 10.2 Пользователь смотрит топ монеты

Пользователь открывает раздел "Топ монеты", выбирает монету из списка и смотрит ее график.

### 10.3 Пользователь проверяет монету

Пользователь открывает раздел "Проверка монет", находит монету и смотрит предварительную оценку халяльности.

### 10.4 Пользователь смотрит спотовый график

Пользователь открывает раздел "Спотовый график", выбирает монету, меняет настройки графика, смотрит график и при необходимости открывает его на весь экран. Если пользователь вошел в аккаунт, измененные настройки графика сохраняются и восстанавливаются при следующем входе.

### 10.5 Пользователь меняет язык

Пользователь переключает сайт с русского языка на английский или обратно. Интерфейс должен обновиться на выбранный язык.

## 11. Что будет описано в отдельных спеках

В отдельных спеках нужно описать:
- дизайн интерфейса;
- технический стек;
- API и источники данных;
- подробные правила халяль-проверки;
- структуру уроков раздела "Основы Трейдинга";
- регистрацию и личный кабинет;
- поведение TradingView-графика;
- адаптивную верстку для мобильных устройств.
