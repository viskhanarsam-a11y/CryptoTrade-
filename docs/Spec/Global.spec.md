# CryptoTrade Global Specification

## 1. Для кого

CryptoTrade создается для пользователей, которые хотят изучать криптовалюты и трейдинг с нуля, но при этом хотят учитывать халяльность криптоактивов.

Основные группы пользователей:
- новички, которым нужны простые объяснения без сложного трейдерского языка;
- мусульмане, которым важно избегать харам-активов;
- пользователи, которые хотят следить за топовыми криптомонетами;
- пользователи, которым важно видеть риски до самостоятельного решения.

## 2. Для чего

Проект нужен, чтобы пользователь мог в одном месте:
- понять базовые термины криптовалют и трейдинга;
- посмотреть топовые криптомонеты и их базовые показатели;
- увидеть предварительную оценку монеты по халяль-критериям;
- сохранить интересные монеты для наблюдения;
- получить предупреждения о рисках.

CryptoTrade не должен заменять финансового консультанта, религиозного эксперта или личную проверку пользователя. Проект дает образовательную информацию, критерии, источники и объяснения.

## 3. Что это за продукт

CryptoTrade — образовательный веб-сервис про криптовалюты, трейдинг и халяльный подход к выбору криптоактивов.

В продукт входят четыре ключевых направления:
- обзор топовых криптомонет;
- подробная страница монеты с рыночной информацией, графиком, рисками и халяль-статусом;
- раздел "Основы Трейдинга" по криптовалютам и трейдингу;
- список наблюдения для выбранных монет.

Главная ценность продукта — помочь пользователю быстро понять, что представляет собой монета, какие у нее риски и почему она может быть помечена как `Halal-friendly`, `Not halal` или `Unknown`.

## 4. Где используется

Проект используется как веб-сайт, доступный с desktop, tablet и mobile.

Основные места использования:
- главная страница с обзором топовых монет;
- страница конкретной монеты;
- раздел "Основы Трейдинга";
- список наблюдения пользователя.

На первом этапе данные пользователя можно хранить локально в браузере. Регистрация, личный кабинет, админ-панель, уведомления, расширенная аналитика и мобильное приложение описываются в отдельных спеках.

## 5. Границы этой спеки

Эта спека отвечает только на верхнеуровневые вопросы:
- для кого создается продукт;
- для чего он нужен;
- что представляет собой продукт;
- где пользователь будет его использовать.

Подробные требования по MVP, интерфейсу, источникам данных, ролям, халяль-анализу, обучению, графикам и техническому стеку должны быть вынесены в отдельные спеки.
