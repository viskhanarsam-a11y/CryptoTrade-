# Feature spec: Сохранение данных

## 1. Назначение

Сохранение данных нужно, чтобы пользователь не терял прогресс, настройки и выбранный язык.

Эта feature опирается на `docs/Spec/1 Functional.md`, разделы:
- "Регистрация и вход";
- "Переключение языка";
- "Проверка монет";
- "Спотовый график".

## 2. Что сохраняется для зарегистрированного пользователя

Для зарегистрированного пользователя в будущем нужно сохранять:
- прогресс в разделе "Основы Трейдинга";
- избранные монеты;
- пользовательские настройки;
- настройки спотового графика;
- выбранный язык.

## 3. Что можно сохранять локально для гостя

Для незарегистрированного пользователя можно локально сохранять:
- выбранный язык;
- последний результат проверки монеты;
- настройки спотового графика.

## 4. Правила для настроек графика

Если пользователь вошел в аккаунт, настройки графика сохраняются в аккаунте.

Если пользователь не вошел, настройки можно временно хранить локально в браузере.

После входа локальные настройки можно предложить применить к аккаунту или автоматически перенести.

Локальные настройки не должны перезаписывать уже сохраненные настройки пользователя без понятного правила.

Полноэкранный режим графика не должен включаться автоматически после входа.

## 5. Что проверить

Нужно проверить:
- выбранный язык сохраняется;
- прогресс в разделе "Основы Трейдинга" может сохраняться для зарегистрированного пользователя;
- последний результат проверки можно временно сохранить;
- настройки графика сохраняются для зарегистрированного пользователя;
- настройки графика можно временно сохранить для гостя;
- после входа локальные настройки не перезаписывают аккаунт неожиданно.
