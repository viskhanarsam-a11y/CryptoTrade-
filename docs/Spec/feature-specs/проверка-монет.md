# Feature spec: Проверка монет

## 1. Назначение

Раздел "Проверка монет" нужен, чтобы пользователь мог найти монету и увидеть предварительную оценку ее халяльности.

Эта feature опирается на `docs/Spec/1 Functional.md`, раздел "Проверка монет".

## 2. Основной сценарий

Пользователь:
- открывает раздел "Проверка монет";
- вводит название или тикер монеты;
- выбирает найденную монету;
- запускает проверку;
- видит результат проверки.

## 3. Что показывает результат проверки

Результат проверки должен показывать:
- название монеты;
- статус халяльности;
- короткое объяснение статуса;
- возможные риски;
- источники данных, если они доступны.

## 4. Возможные статусы

Возможные статусы:
- `Halal-friendly`;
- `Not halal`;
- `Unknown`.

## 5. Цвета статусов

Цвета должны быть строгими:
- `Halal-friendly` всегда зеленый;
- `Not halal` всегда красный;
- `Unknown` всегда нейтральный серый.

Сайт не должен путать цвет и статус.

## 6. Сообщение для Unknown

Для статуса `Unknown` сайт должен показывать сообщение:

`Статус этой монеты пока неизвестен. Мы сообщим больше, когда появится проверенная информация.`

## 7. Загрузка и сохранение результата

Если проверка занимает время, сайт должен показывать индикатор загрузки.

Последний результат проверки можно временно сохранять, чтобы пользователь не потерял его при обновлении страницы.

## 8. Важное ограничение

Сайт не должен выдавать религиозное заключение.

Сайт показывает предварительную оценку и объясняет, почему монета может попадать в конкретный статус.

## 9. Что проверить

Нужно проверить:
- поиск работает по названию монеты;
- поиск работает по тикеру монеты;
- пользователь может выбрать найденную монету;
- пользователь может запустить проверку;
- во время проверки показывается индикатор загрузки;
- результат показывает название монеты;
- результат показывает статус;
- результат показывает объяснение;
- результат показывает риски;
- цвета статусов не перепутаны;
- для `Unknown` показывается правильное сообщение;
- сайт не называет результат религиозным заключением.
