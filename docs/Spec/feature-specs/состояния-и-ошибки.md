# Feature spec: Состояния и ошибки

## 1. Назначение

Эта спека описывает общие состояния, которые должны учитываться в разных features.

Она опирается на `docs/Spec/1 Functional.md`, где описаны загрузка проверки, отсутствие данных цены, сохранение результата и поведение mobile.

## 2. Общие состояния

В разных разделах сайта могут быть состояния:
- загрузка;
- пустой список;
- данные доступны;
- данные недоступны;
- ошибка;
- пользователь не вошел;
- пользователь вошел;
- mobile-меню открыто;
- mobile-меню закрыто.

## 3. Состояния для топ монет

Нужно учитывать:
- список монет загружается;
- список монет доступен;
- текущая цена доступна;
- текущая цена недоступна;
- изменение цены доступно;
- изменение цены недоступно.

## 4. Состояния для проверки монет

Нужно учитывать:
- пользователь еще ничего не ввел;
- пользователь вводит название или тикер;
- монеты найдены;
- монеты не найдены;
- проверка выполняется;
- результат доступен;
- статус `Unknown`;
- произошла ошибка проверки.

## 5. Состояния для графика

Нужно учитывать:
- график загружается;
- график доступен;
- данные графика недоступны;
- обычный режим графика;
- полноэкранный режим графика;
- настройки сохранены;
- настройки не удалось сохранить.

## 6. Правила

Если данные недоступны, сайт не должен ломаться.

Если действие занимает время, сайт должен показывать загрузку.

Если произошла ошибка, пользователь должен видеть понятное сообщение.

На mobile ошибки и загрузка не должны ломать верстку.

## 7. Что проверить

Нужно проверить:
- загрузка отображается там, где действие занимает время;
- отсутствие данных не ломает интерфейс;
- ошибки написаны понятным текстом;
- статусы проверки монет отображаются правильными цветами;
- mobile-версия не ломается в состояниях ошибки и загрузки.
