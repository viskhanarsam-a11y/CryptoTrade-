# Technical spec: Безопасность и правовые ограничения

## 1. Назначение

Эта спека описывает технические и продуктовые ограничения, которые защищают пользователя и проект.

## 2. Финансовое ограничение

CryptoTrade не должен обещать доходность и не должен давать персональные инвестиционные рекомендации.

В интерфейсе нужно избегать формулировок:
- "покупай";
- "продавай";
- "гарантированная прибыль";
- "без риска";
- "точный сигнал".

## 3. Религиозное ограничение

CryptoTrade не должен выдавать фатву.

Правильная формулировка:
- предварительная оценка;
- образовательная информация;
- объяснение критериев;
- источники для самостоятельной проверки.

## 4. Защита аккаунта

Нужно предусмотреть:
- безопасное хранение паролей на backend/BaaS стороне;
- защиту сессий;
- rate limit на вход;
- понятные ошибки без раскрытия лишней информации;
- logout.

## 5. Защита данных

Нельзя хранить чувствительные данные без необходимости.

Минимальные персональные данные для MVP:
- email;
- id пользователя;
- пользовательские настройки.

## 6. Внешние API

API keys нельзя хранить в frontend-коде, если ключ дает доступ к платному или приватному API.

Такие запросы должны идти через backend или serverless function.

## 7. Что надо определить глубже

Нужно уточнить:
- текст дисклеймера;
- нужна ли страница Terms/Privacy;
- какие данные пользователя реально собираются;
- как удалять аккаунт;
- как логировать ошибки без утечки персональных данных.
