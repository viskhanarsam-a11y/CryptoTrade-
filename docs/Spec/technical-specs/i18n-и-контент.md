# Technical spec: i18n и контент

## 1. Назначение

Эта спека описывает перевод интерфейса и хранение обучающих материалов.

## 2. Языки

Проект поддерживает:
- русский;
- английский.

Выбранный язык должен сохраняться:
- локально для гостя;
- в аккаунте для зарегистрированного пользователя.

## 3. Переводы

Все тексты интерфейса должны храниться через ключи переводов.

Нельзя смешивать русский и английский текст прямо внутри компонентов без i18n-слоя.

Примеры групп ключей:
- `navigation`;
- `auth`;
- `coins`;
- `halalCheck`;
- `learning`;
- `errors`;
- `chart`;
- `common`.

## 4. Обучающий контент

Для раздела "Основы Трейдинга" нужно определить формат контента:
- markdown;
- JSON;
- CMS;
- база данных.

Для MVP можно хранить уроки как markdown или JSON-файлы в проекте.

Каждый урок должен иметь:
- `lessonId`;
- `title`;
- `description`;
- `content`;
- `language`;
- `videoUrl`;
- `videoEmbedUrl`;
- `videoLanguage`;
- `order`;
- `isPublished`.

## 5. Видео

Если видео не добавлено:
- UI не должен показывать сломанный видеоблок;
- урок должен оставаться доступным;
- можно показывать только текстовый материал.

Если видео добавлено:
- для раздела "Основы Трейдинга" нужно использовать русскоязычные видео;
- значение `videoLanguage` должно показывать язык видео;
- видео на другом языке нельзя считать готовым к публикации в уроке для русскоязычной аудитории, если пользователь явно не разрешил исключение.

## 6. Что надо определить глубже

Нужно решить:
- библиотеку i18n;
- формат хранения уроков;
- где хранить видео;
- нужен ли CMS;
- как проверять полноту переводов;
- как обновлять контент без релиза приложения.
