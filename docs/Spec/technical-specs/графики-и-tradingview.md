# Technical spec: Графики и TradingView

## 1. Назначение

Эта спека описывает реализацию графиков монет в разделах "Топ монеты" и "Спотовый график".

## 2. Варианты реализации

Вариант 1: TradingView widget.

Плюсы:
- быстро запустить;
- знакомый вид;
- не нужно самостоятельно рисовать свечи.

Минусы:
- меньше контроля;
- зависимость от стороннего виджета;
- возможны ограничения кастомизации.

Вариант 2: собственный график на lightweight-charts.

Плюсы:
- больше контроля над UI;
- можно использовать свои свечные данные;
- легче встроить свои состояния и настройки.

Минусы:
- нужно получать и нормализовать свечи;
- больше ответственности за качество графика.

## 3. Решение для MVP

Для MVP используется TradingView widget, потому что главная цель - быстро получить знакомый пользователю график в стиле TradingView.

`lightweight-charts` можно рассматривать позже, если проекту понадобятся собственные индикаторы, обучение на графике и сохранение точного состояния графика.

## 4. Настройки графика

Нужно поддержать:
- выбранную монету;
- торговую пару;
- таймфрейм;
- тип графика;
- полноэкранный режим;
- тему;
- масштаб и положение, если библиотека это позволяет.

Полноэкранный режим не должен сохраняться как автоматическое состояние после входа.

## 5. Таймфреймы

Минимальный набор:
- 15m;
- 1h;
- 4h;
- 1d;
- 1w.

Если источник данных не поддерживает таймфрейм, UI должен скрывать его или показывать понятную ошибку.

## 6. Mobile

На mobile график должен:
- помещаться по ширине;
- не создавать горизонтальный скролл страницы;
- иметь отдельную кнопку полноэкранного режима;
- не перекрывать навигацию;
- корректно закрываться из fullscreen.

## 7. Что надо определить глубже

Нужно определить:
- fallback при недоступности TradingView widget;
- список поддерживаемых таймфреймов;
- формат chart settings;
- правила сохранения настроек;
- поведение при отсутствии графика.
