# User story spec: История пользователя

## 1. Назначение

Эта спека описывает основной путь пользователя по сайту CryptoTrade.

Она связывает уже покрытые features:
- главная страница;
- навигация;
- топ монеты;
- проверка монет;
- спотовый график;
- Основы Трейдинга;
- регистрация и вход;
- переключение языка;
- сохранение данных;
- состояния и ошибки;
- мобильная версия.

Спека не добавляет новые функции. Она описывает, какие действия пользователь совершает в рамках уже описанных разделов.

## 2. Кто пользователь

Пользователь CryptoTrade может быть:
- новичком, который изучает криптовалюты и трейдинг;
- пользователем, который хочет посмотреть топ монет;
- пользователем, который хочет открыть график монеты;
- пользователем, который хочет увидеть предварительную халяль-оценку монеты;
- гостем без аккаунта;
- зарегистрированным пользователем.

## 3. Первый вход на сайт

Пользователь открывает CryptoTrade и попадает на главную страницу.

На главной странице пользователь должен понять:
- что это сайт CryptoTrade;
- что сайт помогает изучать трейдинг;
- что сайт показывает топ монеты;
- что сайт позволяет смотреть графики;
- что сайт позволяет проверять монеты по халяль-критериям.

С главной страницы пользователь может:
- открыть топ монеты;
- открыть проверку монет;
- открыть спотовый график;
- открыть раздел "Основы Трейдинга";
- открыть регистрацию или вход;
- переключить язык.

## 4. Сценарий: пользователь смотрит топ монет

Пользователь:
- открывает раздел "Топ монеты";
- видит список из 20 монет;
- смотрит название, тикер, логотип, цену и изменение цены;
- видит рост цены зеленым цветом;
- видит падение цены красным цветом;
- нажимает на строку выбранной монеты;
- открывает график этой монеты.

Топ-20 монет должен отображаться компактными строками, а не большими карточками.

Если цена или изменение цены недоступны, пользователь видит нейтральное состояние, а строка монеты не ломается.

## 5. Сценарий: пользователь открывает график

Пользователь может открыть график двумя способами:
- через раздел "Топ монеты";
- через раздел "Спотовый график".

Пользователь:
- выбирает монету;
- видит график выбранной монеты;
- меняет доступные настройки графика, если они реализованы;
- открывает график на весь экран;
- закрывает полноэкранный режим.

Если пользователь вошел в аккаунт, настройки графика могут сохраняться в аккаунте.

Если пользователь не вошел, настройки графика можно временно сохранить локально в браузере.

Полноэкранный режим не должен включаться автоматически после входа.

## 6. Сценарий: пользователь проверяет монету

Пользователь:
- открывает раздел "Проверка монет";
- вводит название или тикер монеты;
- выбирает найденную монету;
- запускает проверку;
- видит результат проверки.

Результат проверки показывает:
- название монеты;
- статус халяльности;
- короткое объяснение;
- возможные риски;
- источники данных, если они доступны.

Возможные статусы:
- `Halal-friendly`;
- `Not halal`;
- `Unknown`.

Цвета статусов должны быть строгими:
- `Halal-friendly` — зеленый;
- `Not halal` — красный;
- `Unknown` — нейтральный серый.

Сайт не должен выдавать религиозное заключение. Он показывает предварительную оценку и объяснение.

## 7. Сценарий: пользователь изучает Основы Трейдинга

Пользователь:
- открывает раздел "Основы Трейдинга";
- видит список тем;
- выбирает тему;
- открывает урок;
- читает материал или смотрит видео, если оно добавлено;
- возвращается к списку уроков.

Раздел "Основы Трейдинга" должен быть доступен без обязательной регистрации.

Если пользователь зарегистрирован, сайт может показывать статус прохождения.

Если видео не добавлено, сайт не должен показывать сломанный видеоблок.

## 8. Сценарий: пользователь меняет язык

Пользователь:
- открывает переключатель языка;
- выбирает русский или английский язык;
- видит интерфейс на выбранном языке;
- обновляет страницу;
- остается на выбранном языке.

Если пользователь вошел в аккаунт, выбранный язык сохраняется для пользователя.

Если пользователь не вошел, выбранный язык можно сохранить локально в браузере.

## 9. Сценарий: пользователь регистрируется или входит

Пользователь может:
- открыть регистрацию;
- создать аккаунт;
- открыть вход;
- войти в аккаунт;
- выйти из аккаунта.

После входа пользователь должен понимать, что он вошел в аккаунт.

После выхода сайт должен показывать состояние гостя.

Регистрация нужна, чтобы в будущем сохранять:
- прогресс в разделе "Основы Трейдинга";
- избранные монеты;
- пользовательские настройки;
- настройки спотового графика;
- выбранный язык.

Ошибки регистрации и входа должны быть понятными.

## 10. Сценарий: пользователь работает с ошибками и загрузкой

Если действие занимает время, пользователь видит состояние загрузки.

Загрузка может быть в разделах:
- топ монеты;
- проверка монет;
- спотовый график;
- раздел "Основы Трейдинга", если материалы загружаются не сразу.

Если данные недоступны, сайт не должен ломаться.

Пользователь должен видеть понятное сообщение, если:
- список монет не загрузился;
- цена монеты недоступна;
- изменение цены недоступно;
- монета не найдена;
- проверка монеты завершилась ошибкой;
- график недоступен;
- настройки не удалось сохранить.

Временная ошибка API не должна менять халяль-статус монеты.

## 11. Сценарий: пользователь использует сайт на телефоне

На mobile пользователь должен иметь возможность:
- открыть и закрыть меню;
- перейти в любой основной раздел;
- посмотреть топ монет строками;
- нажать на строку монеты;
- открыть график;
- открыть и закрыть полноэкранный график;
- проверить монету;
- пройти раздел "Основы Трейдинга";
- переключить язык;
- войти или выйти.

Mobile-версия должна работать без горизонтального скролла.

Текст, кнопки, строки монет, ошибки и загрузка не должны ломать верстку.

## 12. Что проверить

Нужно проверить:
- пользователь понимает назначение сайта на главной странице;
- с главной страницы доступны основные разделы;
- навигация работает на desktop и mobile;
- топ-20 монет отображается строками;
- строки монет кликабельны;
- зеленый и красный цвета изменения цены не перепутаны;
- выбранная монета открывает график;
- график можно открыть и закрыть на весь экран;
- проверка монет показывает правильные статусы и цвета;
- сайт не выдает религиозное заключение;
- раздел "Основы Трейдинга" открывает темы и уроки;
- язык переключается и сохраняется;
- пользователь может зарегистрироваться, войти и выйти;
- ошибки показываются понятным текстом;
- состояния загрузки видны пользователю;
- на mobile нет горизонтального скролла;
- mobile-версия не ломает основные сценарии.
